# Packaging — 최종 조립 (하위 스킬 4/4)

> 호출 주체: `sermon-image-design` 마스터. 입력: 검증 통과 이미지 + 매핑표 확정판 + 검증 판정표.

## 임무

검증을 통과한 이미지를 **예배 현장에서 바로 쓰는 형태**로 조립한다.

## 0. 의존성 점검 (실패해도 전체는 계속)

```bash
python3 -c "import pptx" 2>/dev/null || pip3 install --user python-pptx
which ffmpeg || brew install ffmpeg        # 실패 시 mp4 스킵, 보고에 명시
```

| 의존성 | 실패 시 |
|--------|---------|
| python-pptx | PPTX 스킵, 나머지 산출 + 보고 |
| ffmpeg | mp4 스킵, 나머지 산출 + 보고 |

## 1. 설교용 PPTX (python-pptx)

- 16:9 (13.333×7.5 in), 슬라이드 구성: **표지**(설교 제목·본문 구절) → **장면 슬라이드 ×N** → **마무리**(기도 문구 자리).
- 장면 슬라이드: 이미지 풀블리드 배치 + **별도 텍스트박스**(상단)에 장면 제목·핵심구절.
  이미지에는 글자가 없으므로 모든 문구는 이 레이어에만 — 교사가 자유롭게 수정 가능해야 한다.
- 텍스트박스 규격: 굵은 고딕 계열(시스템 폰트), 36pt 이상, 흰 글자 + 반투명 어두운 바탕(원거리 가독).
- 파일명: `<설교제목>_설교.pptx`.

## 2. Ken Burns 모션 mp4 (ffmpeg)

**장면별 클립** (각 12–20초 — 미취학 설교 이미지 노출 기준):
```bash
ffmpeg -loop 1 -i jpg/01_장면명.jpg -vf \
  "scale=8000:-1,zoompan=z='zoom+0.0008':x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)':d=<초×25>:s=1280x720:fps=25" \
  -t <초> -c:v libx264 -pix_fmt yuv420p clips/01.mp4
```
- 줌 속도 0.0008 수준의 **아주 느린 줌**(어지러움 방지). 장면 분위기에 따라 줌인/줌아웃/팬 교차.
- 초점 좌표(x,y)는 장면 주인공 위치로 지정.

**통합본**: 장면별 클립을 1초 크로스페이드(xfade)로 연결 → `<설교제목>_모션.mp4`. **음원 없음**(무음 트랙).

## 3. 효과명세서 (제안 문서 — 구현 아님)

```markdown
# 효과명세 — <설교 제목> (미취학)
> 음원·연출은 현장 재량. 아래는 제안이며 파일에는 포함되지 않음.

## 01_장면명 (12초)
- BGM 제안: <무드 — 예: 밝고 잔잔한 동요풍, 글로켄슈필 톤>
- 효과음 제안: <1–2초 단발 — 예: 양 울음 "메에~">
- 모션(통합 mp4 적용): <예: 천천히 줌인, 주인공 초점>
- 전환: <예: 부드러운 페이드>
```
- 공통 원칙: 볼륨 작게, 효과음 1–2초 단발, 과자극·큰 소리 금지.

## 4. 폴더 정리 + 완료 체크리스트

```
~/Desktop/Agent/<프로젝트명>/
├── jpg/            NN_장면명.jpg (최종 이미지)
├── svg/            (SVG 경로 사용 시 원본)
├── prompts/        장당 프롬프트·라운드 기록
├── clips/          장면별 모션 클립
├── <제목>_설교.pptx
├── <제목>_모션.mp4
├── 매핑표.md        (확정판 — 검증 라운드 반영)
└── 효과명세.md
```

마스터에 반환할 완료 체크리스트:
- [ ] 이미지 N장 (≤12) 전부 1280×720 JPG, 일관 화풍
- [ ] 매핑표 확정판 (장면↔설교문 단락 대응)
- [ ] PPTX (또는 스킵 사유)
- [ ] 장면별 클립 + 통합 mp4 (또는 스킵 사유)
- [ ] 효과명세서
- [ ] prompts/ 재생성 기록 포함
- [ ] 검증 잔여 ⚠️ 플래그 전달
