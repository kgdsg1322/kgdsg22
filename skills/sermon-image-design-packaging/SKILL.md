# Packaging — 최종 조립 (하위 스킬 4/4)

> 호출 주체: `sermon-image-design` 마스터. 입력: 검증 통과 이미지 + 매핑표 확정판 + 검증 판정표.

## 임무

검증을 통과한 이미지를 **예배 현장에서 바로 쓰는 형태**로 조립한다.

## 0. 의존성 점검 (실패해도 전체는 계속)

```bash
python3 -c "import pptx" 2>/dev/null || pip3 install --user python-pptx
which ffmpeg || brew install ffmpeg        # 실패 시 mp4 스킵, 보고에 명시
```

| 의존성 | 실패 시 |
|--------|---------|
| python-pptx | PPTX 스킵, 나머지 산출 + 보고 |
| ffmpeg | mp4 스킵, 나머지 산출 + 보고 |

## 1. 설교용 PPTX — 결정론 스크립트 (코드 신작성 금지)

```bash
python3 <scripts_dir>/build_pptx.py <프로젝트폴더> <프로젝트폴더>/매핑표.md --title <설교제목> --verse <본문구절>
```
- 구성·규격은 스크립트가 보장: 16:9, 표지 → 장면 ×N(풀블리드 이미지 + 상단 텍스트박스, 교사 수정 가능) → 마무리,
  슬라이드 수(N+2) 자동 검증.
- `SKIP`(python-pptx 없음) 출력 시 `pip3 install --user python-pptx` 후 1회 재실행, 그래도 실패면 PPTX만 스킵·보고.

## 2. Ken Burns 모션 mp4 — 결정론 스크립트 (코드 신작성 금지)

```bash
python3 <scripts_dir>/build_motion.py <프로젝트폴더> <프로젝트폴더>/매핑표.md --title <설교제목> --sec 15
```
- 스크립트가 보장: 장면별 클립(기본 15초, `--sec` 12–20 권장), 저속 줌 0.0008/frame(어지러움 방지)·줌인/줌아웃 교차,
  1초 크로스페이드 통합본 `<설교제목>_모션.mp4`, **음원 없음**, 클립·통합본 길이 실측 검증.
- `SKIP`(ffmpeg 없음) 출력 시 `brew install ffmpeg` 1회 시도 후 재실행, 그래도 실패면 mp4만 스킵·보고.

## 3. 효과명세서 (제안 문서 — 구현 아님)

```markdown
# 효과명세 — <설교 제목> (미취학)
> 음원·연출은 현장 재량. 아래는 제안이며 파일에는 포함되지 않음.

## 01_장면명 (15초 — build_motion --sec 값과 동일하게 기재)
- BGM 제안: <무드 — 예: 밝고 잔잔한 동요풍, 글로켄슈필 톤>
- 효과음 제안: <1–2초 단발 — 예: 양 울음 "메에~">
- 모션(통합 mp4 적용): <예: 천천히 줌인, 주인공 초점>
- 전환: <예: 부드러운 페이드>
```
- 공통 원칙: 볼륨 작게, 효과음 1–2초 단발, 과자극·큰 소리 금지.

## 4. 폴더 정리 + 완료 체크리스트

```
~/Desktop/Agent/<프로젝트명>/
├── jpg/            NN_장면명.jpg (최종 이미지)
├── svg/            (SVG 경로 사용 시 원본)
├── prompts/        장당 프롬프트·라운드 기록
├── clips/          장면별 모션 클립
├── <제목>_설교.pptx
├── <제목>_모션.mp4
├── 매핑표.md        (확정판 — 검증 라운드 반영)
└── 효과명세.md
```

마스터에 반환할 완료 체크리스트:
- [ ] 이미지 N장 (≤12) 전부 1280×720 JPG, 일관 화풍
- [ ] 매핑표 확정판 (장면↔설교문 단락 대응)
- [ ] PPTX (또는 스킵 사유)
- [ ] 장면별 클립 + 통합 mp4 (또는 스킵 사유)
- [ ] 효과명세서
- [ ] prompts/ 재생성 기록 포함
- [ ] 검증 잔여 ⚠️ 플래그 전달
