# Verification — 검증 게이트 (하위 스킬 3/4)

> 호출 주체: `sermon-image-design` 마스터(가능하면 독립 에이전트로 실행). 입력: 매핑표 + 설교문 원문 + 이미지 전량 + prompts/.

## 임무

이 패키지가 어린 영혼과 강단에 나가기 전 **마지막 관문**이다. 과장·거짓 확신·날조를 여기서 끊는다.
판정은 반드시 **이미지를 Read 도구로 직접 열어 실측**으로 내린다. 파일명·프롬프트만 보고 추측 판정 금지.

## 5항목 게이트

`references/verification-checklist.md` 전문을 Read하여 적용한다. 요약:

| 항목 | 검증 내용 | FAIL 예시 |
|------|-----------|-----------|
| **V1 설교문 정합** | 모든 이미지가 매핑표의 설교문 원문 인용과 실제 대응하는가. 설교문에 없는 인물·사건·소품이 그려지지 않았는가 | 설교문에 없는 천사 등장 |
| **V2 신학·본문 정합** | 성경 본문이 명시하지 않은 사항을 단정 형상화하지 않았는가. 정통 기독교(개혁주의) 기준에서 오해 소지가 없는가 | 선악과를 사과로 특정 |
| **V3 아동 안전** | 1–7세 기준 공포·위협·피·어둠·과자극 요소가 없는가. 주인공이 밝고 안정적인가 | 어두운 골짜기에 홀로 우는 아이 |
| **V4 저작권 안전** | prompts/ 기록에 출판물·작가·브랜드 고유명사가 없는가. 기존 작품의 식별 가능한 복제가 아닌가 | 프롬프트에 "○○바이블 스타일" |
| **V5 시리즈 일관성** | 동일 인물이 전 장면 동일 외형(비율·머리색·의상)인가. 팔레트·선·질감이 통일됐는가 | 3번 양만 귀 모양 다름 |

## 판정표 형식

```markdown
# 검증 판정표 — 라운드 <N>

| 이미지 | V1 | V2 | V3 | V4 | V5 | 종합 |
|--------|----|----|----|----|----|------|
| 01_장면명.jpg | ✅ | ✅ | ✅ | ✅ | ✅ | PASS |
| 03_장면명.jpg | ✅ | ❌ | ✅ | ✅ | ✅ | FAIL |

## FAIL 근거 (실측 기술)
- 03 / V2: <이미지에서 직접 본 것> — <왜 문제인가>

## 수정 지시서
- 03번: <구체적으로 무엇을 어떻게 바꿀지 — 재생성 프롬프트에 반영 가능한 수준으로>
```

## 운영 규칙

1. **전수 검사.** 표본 검사 금지. 모든 이미지 × 모든 항목.
2. **근거 없는 PASS 금지.** 각 판정에 실측 근거가 따라야 한다. 확인 불가 항목은 PASS가 아니라 "미확인"으로 보고.
3. **재생성 최대 3라운드.** 라운드마다 FAIL 이미지만 재검증하되 V5(일관성)는 전체 대조로 재확인.
4. **3라운드 후 잔여 FAIL** → ⚠️ 플래그 + 사유 + 수동 확인 권고로 마스터에 보고. 무한 반복 금지.
5. **검증자 분리.** 본 스킬 실행 컨텍스트는 생성 컨텍스트와 분리(독립 에이전트)가 원칙. 불가피하게 같은
   컨텍스트면 그 사실을 판정표에 명시한다.
6. 설교문 자체의 신학 오류가 의심되면 — 이미지 스킬의 권한 밖이므로 고치지 말고 ⚠️ 플래그로만 보고한다.
